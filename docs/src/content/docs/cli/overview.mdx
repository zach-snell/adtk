adtk provides a full-featured command-line interface for Azure DevOps. All commands support a `--json` flag for raw JSON output, making them composable with other tools.

## Authentication

Before using any command, authenticate with `adtk auth` or set the `AZURE_DEVOPS_ORG` and `AZURE_DEVOPS_PAT` environment variables.

## Global Flags

| Flag | Description |
|------|-------------|
| `--json` | Output raw JSON instead of formatted tables |
| `--version` | Print adtk version |

| Command | Aliases | Description |
|---------|---------|-------------|

| [`status`](/cli/auth/) | | Show current authentication status |

| [`pull-requests`](/cli/pull-requests/) | `pr`, `prs` | Manage pull requests (list, get, reviewers) |
| [`pipelines`](/cli/pipelines/) | `pipe` | Manage CI/CD pipelines (list, get, runs, definitions, var-groups, environments) |
| [`wiki`](/cli/wiki/) | | Manage wiki pages (list, get, pages) |

| [`iterations`](/cli/iterations/) | `iter`, `sprints` | Manage iterations/sprints (list, current, settings) |

| [`projects`](/cli/projects/) | `proj` | Manage projects and teams (list, get, teams) |
| [`attachments`](/cli/attachments/) | `attach` | Manage work item attachments (list) |
| [`test-plans`](/cli/test-plans/) | `tp` | Manage test plans, suites, and cases |
| [`security`](/cli/security/) | | Manage Advanced Security alerts |

<Aside type="tip">
Most commands that operate within a project accept a `--project` or `-p` flag. Some commands like `projects list` work at the organization level and don't require it.
</Aside>
